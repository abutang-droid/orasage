## 1. 项目定位与愿景

OraSage 是一个依托东方文化语境的内容电商平台，核心模型为：

- 命理服务（八字 / 紫微 / 塔罗）提供“高频内容触点与情绪价值”
- 数字付费产品（深度报告、AI 解读）承接首单变现
- 电商与会员体系承接复购与长期收入

一句话定位：**“命理内容 + 数字服务 + 实物电商”一体化增长平台。**

---

## 2. 当前产品与技术现状（结合项目事实）

### 2.1 已有基础能力

目前仓库已具备 8 应用子域架构与核心分工：

- `main`：主门户与流量入口
- `auth-service`：统一 JWT 认证与用户中心能力
- `shop`：商城与支付逻辑承接
- `admin`：后台骨架（待业务化）
- `cms`：内容管理骨架（待业务化）
- `bazi` / `ziwei` / `tarot`：三条命理服务产品线

并且具备：

- 跨应用登录桥接（共享 cookie / JWT）
- 子域应用边界清晰，利于分层演进
- 命理服务到交易转化的基础链路

### 2.2 商业化前的关键缺口

需要优先补齐的部分：

1. 真实 VPS 全链路验收（DNS / SSL / Nginx / systemd / DB）
2. `admin` / `cms` 从骨架升级到“最小可运营”
3. 核心链路 E2E（登录 -> 测算 -> 下单 -> 支付）
4. 监控告警与可观测性体系（支付成功率 / 延迟 / 错误率）

## 3. 商业模式（按 Keynote 主线重组）

## 3.1 数字内容付费（高毛利主引擎）

- 深度命理报告（如：八字/紫微深度分析）
- AI 互动解读（即时问答/场景化咨询）
- 场景化测算包（情感、事业、财富、健康）

特点：交付快、可标准化、边际成本低、毛利高。

## 3.2 会员订阅（经常性收入）

- 分级会员（月卡 / 季卡）
- 会员权益：专属内容、解读额度、商城折扣、活动优先权
- 目标：提升留存与复购，形成 ARR 基盘

## 3.3 实物电商（客单与利润空间）

- 五行手串 / 文创套装 / 节气主题组合
- 与数字产品联动捆绑，提高转化效率
- 联名与限定款扩大品牌溢价

## 3.4 高客单服务（品牌壁垒）

- 年度运势规划 / 深度咨询
- 高净值用户定制服务
- 用于建立信任与专家型品牌资产

## 4. 增长与转化漏斗（5人团队可执行）

## 4.1 流量入口

- SEO：覆盖高频问题词与场景词
- 短视频矩阵：TikTok / FB / 小红书（按实际渠道选择）
- 社交裂变：私域社群与转介绍机制

## 4.2 转化路径

免费体验 -> 低门槛首购（如 9.9） -> 深度服务升级 -> 会员订阅 -> 电商复购

## 4.3 关键运营机制

- A/B 测试：价格、首屏文案、推荐位
- 客服 SOP：首问响应、咨询分层、标准化交付
- 数据看板：漏斗转化、CAC、LTV、支付成功率

## 5. 5人最小化组织配置（MVO）

### 角色配置

1. **负责人（1）**：商业策略、目标管理、跨团队协同
2. **技术负责人（1）**：架构稳定性、上线流程、数据口径
3. **全栈工程师（1）**：业务迭代、支付链路、后台最小功能
4. **增长运营（1）**：内容分发、渠道测试、漏斗优化
5. **内容/交付（1）**：命理内容生产、会员服务、客服交付

### 协作原则

- 一人双责，先保证闭环，再做扩张
- 每周看经营指标，不以“功能数量”作为成功标准
- 工程迭代围绕收入与稳定性优先级推进

## 8. 财务模型（5人团队基准版）

> 单位：CNY，以下为经营测算模板，需按真实转化率动态校准。

## 8.1 成本结构（月度）

- 人力成本（5人）：120,000 ~ 180,000
- 技术与云资源：10,000 ~ 20,000
- 增长投放：20,000 ~ 80,000（先小后大）
- 内容与运营杂支：10,000 ~ 20,000

**月总成本区间：160,000 ~ 300,000**

## 8.2 收入结构（月度毛利口径）

- 数字内容毛利 = 数字产品收入 × 数字产品毛利率
- 会员毛利 = 会员收入 × 会员毛利率
- 电商毛利 = 电商 GMV × 商品毛利率

**总毛利 = 数字毛利 + 会员毛利 + 电商毛利**

## 8.3 三情景（示例）

| 情景 | 月毛利 | 月成本 | 月经营结果 |
|---|---:|---:|---:|
| 保守 | 140,000 | 200,000 | -60,000 |
| 基准 | 240,000 | 220,000 | +20,000 |
| 进取 | 380,000 | 260,000 | +120,000 |

现金建议：准备 9 个月运行缓冲，滚动复盘投放 ROI。

## 9. KPI 体系与里程碑

## 9.1 增长与转化

- 注册转化率（Reg Rate）
- 首购转化率（Trial -> Paid）
- 获客成本（CAC）
- 每付费用户收入（ARPPU）

## 9.2 经营效率

- 生命周期价值（LTV）
- 回本周期（Payback）
- 会员渗透率
- 电商复购率

## 9.3 体验与稳定性

- 支付成功率
- 接口错误率
- P95 响应延迟
- 客诉率 / 服务满意度

## 10. 风险与应对策略

1. **支付迁移风险**：先灰度、后全量，建立回滚路径  
2. **稳定性风险**：关键链路监控 + 异常告警 + 值班机制  
3. **投放失控风险**：设置预算闸门，以回本周期驱动扩量  
4. **合规风险**：内容审查、隐私条款、支付与数据合规先行
