# Agent 铁规（改动前必读）

> **每次动手改代码、改 UI、改配置之前，必须先完整阅读本文。**  
> 适用于 Cloud Agent、Cursor Agent 及所有在本仓库提交的协作者。

---

## 铁规

### 1. 不擅自改布局

未经**产品负责人明确确认**，不得改动任何界面布局，包括但不限于：

- 页面结构、区块顺序、栅格与留白
- 导航、底栏、侧栏、页眉页脚的位置或组成
- 组件在页面中的摆放、对齐、尺寸层级
- 响应式断点下的版面重组

允许在**已确认的任务范围内**做与布局无关的修复（例如修 bug、对接 API、改文案错别字），但若修复会连带改变视觉或结构，须先说明并获确认。

### 2. 不主动增删功能

未经**产品负责人明确确认**，不得：

- **新增**用户可见或可调用的功能、入口、流程步骤、配置项
- **删除或下线**现有功能、入口、API、页面或交互

任务若只要求修 bug、补测试、改后端逻辑，应严格限定在该范围内；发现「顺便可以做」的优化或重构，须先列出方案，等确认后再做。

---

## 动手前检查清单

在提交 PR 或推送改动前，自问：

- [ ] 本次是否涉及布局或视觉结构？若是，是否已获确认？
- [ ] 本次是否新增或移除了功能？若是，是否已获确认？
- [ ] 改动范围是否与用户/任务描述一致，没有擅自扩大？

若有任一项为「是」但未获确认，**停止提交**，先说明计划并等待确认。
