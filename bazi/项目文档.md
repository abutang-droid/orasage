**品牌**: Orasage (Ora + Sage)  
**技术栈**: React 19 + Express 4 + tRPC 11 + Drizzle ORM + MySQL + Tailwind CSS 4  
**语言**: TypeScript 5.9  
**测试**: Vitest（6 文件 / 54 测试全部通过）

## 1. 项目概述

Orasage 是一个八字（四柱）命理排盘计算器，支持公历/农历输入，可进行单人排盘和双人合盘分析。核心计算引擎在浏览器端完成，后端提供用户认证、AI 解读、历史记录存储。

### 核心功能

| 功能 | 状态 |
|------|------|
| 单人八字排盘 | ✅ 完成 |
| 双人合盘分析 | ✅ 完成 |
| 真太阳时修正（均时差查表） | ✅ 完成 |
| 城市搜索（拼音/中文/英文，592城） | ✅ 完成 |
| 农历-公历互转 | ✅ 完成 |
| AI 解读报告（分节折叠） | ✅ 完成 |
| 五行手串推荐 | ✅ 完成 |
| 今日运势 | ✅ 完成 |
| 计费方案（3档） | ✅ 完成 |
| 历史记录 | ✅ 完成 |

## 4. 数据流

### 排盘流程

```
用户输入 → Home.tsx (handleSubmit)
  → toInput() 转换数据 + getCityCoords() 查询城市坐标
  → calcSingleBazi() / calcDoubleBazi()
    → getLunarData() 查农历干支
    → calcWuXingWeighted() 五行藏干加权
    → analyzeStrength() 身强弱分析
    → determineFavorable() 喜用神/忌神
    → calcDaYunNew() 大运排列
    → calcShensha() 神煞
  → SingleBaziResultView 展示结果
```

### 支付流程

```
结果页计费墙 → 选择方案（basic/advanced/premium）
  → handlePlanSelect() 解锁内容
  → 展示完整命盘 + AI解读 + 手串推荐
```

# 配置环境变量 (.env)

# VITE_OAUTH_PORTAL_URL=http://localhost:3000

# 启动
pnpm db:push

## 9. 关键约定

- **tRPC-first**: 后端调用统一使用 `trpc.*.useQuery/useMutation`
- **Auth**: 通过 `isAuthenticated` 判断登录状态
- **Datetime**: DB 存 UTC Unix 时间戳，展示时转本地时区
- **数据类型**: PlanType = `"basic" | "advanced" | "premium"`
- **农历数据**: 本地 `/client/public/data/` 存储 JSON，按十年分文件，文件名有哈希后缀
- **城市数据**: 内嵌在 `cityData.ts` 中（~200 城），无需外部加载

| | 路径 |

> 宿主机目录通过 Docker volume 挂载，容器重建后报告不会丢失。
